# AZDOME — Next.js Reference Storefront

A complete, production-ready DTC storefront for AZDOME dash cams. Built with
Next.js 14 App Router, Tailwind CSS, framer-motion, and TypeScript.

## What's inside

- **24 marketing & content routes** — homepage, PDPs, collections, scenarios,
  legal, support, about, careers, press, affiliate, wholesale, 404
- **Multi-language** — 6 locales (EN / 中文 / 日本語 / DE / FR / ES) with a
  globe switcher in the navbar and footer
- **Real cart system** — context-backed, localStorage-persisted, with cart
  drawer + full cart page + mock checkout
- **PDP polish** — sticky buy box, gallery, breadcrumbs, scroll progress,
  reviews with helpful counter, FAQ accordion, sticky bottom CTA
- **Downloads** — 11 firmware versions + 15 multi-language PDF manuals, all
  real downloadable files generated by a script
- **Admin panel** at `/admin`
  - Image asset management (29 slots across 8 groups)
  - Firmware upload + version management
  - PDF manual upload per model × language
- **User accounts** at `/account/login`, `/account/signup`, `/account`
  - Email + password, scrypt-hashed, HMAC-signed session cookies

## Local development

```bash
npm install
npm run dev
# open http://localhost:3000
```

Default admin password is `admin` (login at `/admin/login`). Override in
`.env.local`:

```bash
cp .env.example .env.local
# then edit ADMIN_PASSWORD and AUTH_SECRET
```

## Regenerate static assets

```bash
# Brand-consistent SVG-rendered images (32 product mockups, scenarios, etc.)
node scripts/generate-images.mjs

# Firmware binaries + multilingual PDF manuals
node scripts/generate-downloads.mjs
```

## Deploy to Vercel

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2FYOUR-USERNAME%2Fazdome-web&env=ADMIN_PASSWORD,AUTH_SECRET&envDescription=Admin%20password%20and%20a%20strong%20random%20auth%20secret)

After pushing this repo to GitHub:

1. Click the **Deploy with Vercel** button above (replace `YOUR-USERNAME`).
2. Vercel auto-detects Next.js. Set `ADMIN_PASSWORD` and `AUTH_SECRET` env
   vars when prompted.
3. First deploy completes in ~90 seconds.

### ⚠ Production caveat — read-only filesystem

The default storage adapter writes to `public/images/` and `public/downloads/`,
which works in local dev but **fails on Vercel/Netlify serverless** (the
runtime filesystem is read-only). For production:

- **Images**: swap `lib/storage.ts`'s `localStorage` adapter for Vercel Blob,
  Cloudflare R2, or S3.
- **User accounts + admin metadata** (`data/db.json`): swap `lib/db.ts` for
  Vercel KV, Upstash Redis, or a real DB (Postgres via Neon/Supabase).

Both files have storage-adapter abstractions designed for this swap —
typically ~10 lines of change.

## Project structure

```
app/
├── layout.tsx              Root: providers, fonts, metadata
├── page.tsx                Homepage
├── products/[slug]/        Dynamic PDP (6 SKUs)
├── collections/[slug]/     Dynamic collection (9 collections)
├── scenarios/[slug]/       Scenario landing (family / rideshare / parking)
├── legal/[slug]/           Privacy / Terms / Warranty / etc.
├── support/                Hub + install + firmware + manuals + contact
├── admin/                  Image / firmware / manual management
├── account/                Sign in / sign up / dashboard
├── api/account/            Session check + logout
├── cart, checkout, ...     E-commerce flow
├── icon.tsx                Generated favicon
└── opengraph-image.tsx     Generated OG share image

components/
├── ui/                     Button, SectionHeader, Logo, EmailCapture, AnimatedCounter
├── admin/                  AdminNav, SlotCard, DownloadsManager
├── Hero, Navbar, Footer    Site chrome
└── ...                     ~30 feature components

lib/
├── products.ts             Single source of truth for SKUs
├── downloads.ts            Firmware + manual catalog (db-overlay-aware)
├── i18n/                   6-language translation dictionaries
├── image-slots.ts          Admin-manageable image slots
├── auth.ts                 User auth (scrypt + HMAC sessions)
├── admin-auth.ts           Admin auth (HMAC-signed token)
├── storage.ts              Storage adapter (swap for cloud)
└── db.ts                   JSON-file KV store (swap for real DB)

scripts/
├── generate-images.mjs     SVG → JPG via sharp
└── generate-downloads.mjs  pdf-lib + binary blobs

public/
├── images/                 Generated brand assets
└── downloads/              Generated firmware + manuals
```
